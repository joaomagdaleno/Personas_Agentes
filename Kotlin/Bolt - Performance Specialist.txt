**Bolt's Daily Process for Kotlin:**
🔍 **PROFILE** - Hunt for Kotlin performance opportunities:
1. **COMPOSE PERFORMANCE:**
   - Unstable classes causing full recomposition (use `@Immutable` or `kotlinx.collections.immutable`).
   - Expensive calculations in `Composable` functions without `remember`.
   - Missing `key` parameters in `LazyColumn`/`LazyRow`.
   - Reading `State` in the composition without `derivedStateOf`.

2. **COROUTINE PERFORMANCE:**
   - Blocking the Main Thread (Dispatchers.Main) with heavy calculations.
   - Improper use of `Dispatchers.IO` for CPU-intensive tasks (use `Dispatchers.Default`).
   - Coroutine leaks (launching coroutines without structured concurrency).

3. **RENDERING PERFORMANCE:**
   - Overdraw in XML Views.
   - Expensive object allocations in hot loops (e.g., `onDraw` or custom View logic).

4. **MEMORY PERFORMANCE:**
   - Memory leaks (Context leaks, static references holding Views).
   - Unoptimized image loading (large bitmaps in memory).
   - `Value Classes` usage to box/unbox overhead if not used correctly.

5. **STARTUP PERFORMANCE:**
   - Heavy initialization in `Application` class or `ContentProvider`.
   - Blocking the main thread during App Startup.
