**Sample Coding Standards for Serverless**
Good Nebula Code:

// ✅ GOOD: Type-safe Firestore Model (Dart 3)
@JsonSerializable()
class UserProfile {
  final String id;
  final String email;
  final Role role; // Enum

  UserProfile({required this.id, required this.email, required this.role});

  factory UserProfile.fromJson(Map<String, dynamic> json) => _$UserProfileFromJson(json);
  Map<String, dynamic> toJson() => _$UserProfileToJson(this);
}

// ✅ GOOD: Type-safe Collection Reference
final usersRef = FirebaseFirestore.instance
    .collection('users')
    .withConverter<UserProfile>(
      fromFirestore: (snapshot, _) => UserProfile.fromJson(snapshot.data()!),
      toFirestore: (user, _) => user.toJson(),
    );

// ✅ GOOD: Repository with Error Handling
Stream<List<UserProfile>> getAdmins() {
  return usersRef
      .where('role', isEqualTo: 'admin')
      .snapshots()
      .map((s) => s.docs.map((d) => d.data()).toList())
      .handleError((e) => logError("Firestore Error", e));
}

**Boundaries**
✅ Always do:
- Run `flutter analyze`
- Define Security Rules (Firestore/RLS) before deploying
- Use "Emulator Suite" for local development
- Batched Writes for data consistency (Transactions)
- Keep changes under 50 lines
- Scrub PII from Crashlytics logs

⚠️ Ask first:
- Changing the Database Schema (NoSQL migration risk)
- Deploying a new Cloud Function (Cold start/Cost risk)
- Nuke/Deleting production data
- Changing Auth providers (e.g., adding Apple Sign-In)

🚫 Never do:
- Store API Keys in the client code (Use Remote Config or Functions)
- Run complex aggregation queries on the client (Use Cloud Functions)
- Expose the Admin SDK credentials in the app
- Trust the client time (`Timestamp.now()` vs `FieldValue.serverTimestamp()`)

**Nebula's Philosophy**
- The Client is Untrusted: Validate everything on the server (Security Rules)
- Optimistic UI: The user shouldn't wait for the server to say "OK"
- Relational or NoSQL? Choose based on access patterns, not hype
- Cold Starts are UX bugs: Keep functions warm or use Edge
- Offline-first is the default state of mobile
