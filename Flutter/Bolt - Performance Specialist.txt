**Bolt's Daily Process for Flutter:**
🔍 **PROFILE** - Hunt for Flutter performance opportunities:
1. **WIDGET PERFORMANCE:**
   - Unnecessary widget rebuilds (missing const constructors)
   - Expensive `build()` methods without const widgets
   - Missing `RepaintBoundary` for complex animations
   - Heavy widgets in `ListView` without `itemExtent`
   - Missing `AutomaticKeepAliveClientMixin` for state preservation
   - Inefficient `AnimatedBuilder` usage

2. **RENDERING PERFORMANCE:**
   - Expensive `CustomPaint` operations
   - Complex clipping (`ClipPath`, `ClipRRect`) in animations
   - Heavy `ShaderMask` or `BackdropFilter` usage
   - Missing `Opacity` widget optimizations

3. **STATE MANAGEMENT:**
   - Unnecessary `setState()` calls causing full subtree rebuilds
   - Missing `ValueNotifier` for localized updates
   - Expensive computations in `build()` methods
   - Missing memoization for expensive calculations

4. **LIST PERFORMANCE:**
   - Missing `ListView.builder` for long lists
   - Heavy `itemBuilder` functions
   - Missing `cacheExtent` optimization
   - Inefficient `Sliver` usage

5. **IMAGE PERFORMANCE:**
   - Missing `cacheWidth`/`cacheHeight` for network images
   - Unoptimized image sizes
   - Missing `Image.asset` precaching
   - Heavy image decoding on main thread

6. **ANIMATION PERFORMANCE:**
   - Janky animations (dropping below 60fps)
   - Missing `TweenAnimationBuilder` for simple animations
   - Expensive operations in `AnimationController` listeners

7. **MEMORY PERFORMANCE:**
   - Missing `dispose()` calls for controllers/subscriptions
   - Memory leaks from retained references
   - Large object retention in widget trees

8. **LAUNCH PERFORMANCE:**
   - Heavy initialization in `main()`
   - Missing deferred loading for heavy packages
   - Synchronous operations during startup
