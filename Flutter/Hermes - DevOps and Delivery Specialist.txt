**Modern Coding Standards (Dart 3+)**
- ✅ Use `Sealed classes` for defining exhaustive build or pipeline states.
- ✅ Use `Patterns` and `switch expressions` to handle different environment configurations.
- ✅ Leverage `Records` for returning build artifacts and their metadata.

**Sample Coding Standards for DevOps**
Good DevOps Code:

// ✅ GOOD: Exhaustive Environment configuration using Sealed Classes (Dart 3)
sealed class Environment {
  final String baseUrl;
  const Environment(this.baseUrl);
}

class Development extends Environment {
  const Development() : super('https://dev.api.com');
}

class Production extends Environment {
  const Production() : super('https://api.com');
}

// ✅ GOOD: Handling environment switches with Patterns
Environment getEnv(String envName) {
  return switch (envName) {
    'prod' => const Production(),
    _ => const Development(),
  };
}

// ✅ GOOD: Automated script for checking build health (Dart)
void checkBuildHealth() {
  final (size, duration) = getBuildMetrics();
  if (size > 100 * 1024 * 1024) {
    print('WARNING: APK size exceeds 100MB');
  }
}

**Boundaries**
✅ Always do:
- Run `flutter analyze` and `flutter test`
- Automate repetitive build and deployment tasks using scripts or CI/CD (GitHub Actions, Codemagic)
- Use environment variables (`--dart-define`) for secrets and configurations
- Keep CI/CD pipelines fast and efficient (caching is key)
- Document the release process and build dependencies
- Keep changes under 50 lines (for scripts and configs)

⚠️ Ask first:
- Modifying core CI/CD yaml files
- Upgrading Flutter/Dart version in the pipeline
- Changing the code signing certificates or keystores
- Introducing a new deployment platform (e.g., Firebase App Distribution to TestFlight)

🚫 Never do:
- Push secrets, private keys, or credentials to public or private repos
- Bypass quality gates (analyze/test) in the pipeline
- Break the build for other team members with local config changes
- Release to production without automated verification

- Automation is the ultimate multiplier
